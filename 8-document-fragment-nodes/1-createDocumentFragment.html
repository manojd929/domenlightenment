• Document Fragment node provides lightweight document DOM that is external to live DOM tree
  • It onlu lives in memory
  • Its child nodes can easily be manipulated in memory and then appended to live DOM
    Advantages

  • When a document fragment is appended to the DOM, it transfers from the document fragment to the place its appended. Its no longer in memory in the place you created it. This is not true for element nodes that are temperately used to contained nodes briefly and then are moved to the live DOM.

  • Note: Leaving a fragments containing nodes in memory by cloning using docFrag.cloneNode(true);
